================================================================================
                            KEY LINQ PATTERNS USED
================================================================================

1. INCLUDE (Eager Loading)
   - Avoids N+1 query problems
   - Example: .Include(d => d.Department)

2. THENINCLUDE (Nested Navigation)
   - Loading multi-level relationships
   - Example: .Include(s => s.Doctor).ThenInclude(d => d.Department)

3. WHERE Filtering
   - Soft delete: !entity.IsDeleted
   - Foreign key filtering: entity.ForeignKeyId == value
   - Enum filtering: entity.Status == LabRequestStatus.Pending
   - Multiple conditions: && operators

4. ORDERBY/ORDERBYDESCENDING
   - Results ordered by name or date
   - Most recent first for time-sensitive data

5. FIRSTORDEFAULT vs ANY
   - FirstOrDefault for single entity retrieval
   - Any for existence checks (more efficient)

6. COMPOSITE KEY QUERIES
   - Multiple WHERE conditions for composite keys
   - Example: Where(x => x.KeyPart1 == val1 && x.KeyPart2 == val2)

================================================================================
                        SOFT DELETE IMPLEMENTATION
================================================================================

All repositories implement soft delete pattern:

- GetAllAsync/GetByXAsync: Filter WHERE !IsDeleted
- SoftDeleteAsync: Set IsDeleted = true, update timestamp
- DeleteAsync: Physical deletion from database
- ExistsAsync: Check !IsDeleted for active records only

All INCLUDE queries respect soft delete by filtering related collections:
  .Include(d => d.Doctors.Where(doc => !doc.IsDeleted))

================================================================================
                          TIMESTAMP MANAGEMENT
================================================================================

- AddAsync: Sets CreatedAt = DateTime.UtcNow, IsDeleted = false
- UpdateAsync: Sets UpdatedAt = DateTime.UtcNow
- SoftDeleteAsync: Sets UpdatedAt = DateTime.UtcNow, IsDeleted = true

================================================================================
                          EXCEPTION HANDLING
================================================================================

- MarkAsReadAsync (Notification): Throws InvalidOperationException if not found
- All Delete methods return bool (false if not found, true if successful)
